## 1. 핵심 가치

### 신뢰 (Trust)
- **비가역적 동작 최소화**: 삭제는 항상 "최근 삭제된 항목"으로 먼저 이동
- **명확한 확인 단계**: 중요한 동작 전 항상 사용자 확인 요청
- **투명한 진행 상태**: 작업 진행률을 명확히 표시

### 명료함 (Clarity)
- **직관적인 시각화**: 문제 유형별 명확한 아이콘과 색상 체계
- **단순한 언어**: 기술 용어 대신 일상 언어 사용
  - ❌ "iCloud 동기화 대기 중인 PHAsset"
  - ✅ "다운로드 필요한 사진"

### 효율 (Efficiency)
- **스마트 추천**: AI 기반 삭제 추천으로 결정 피로 감소
- **일괄 처리**: 유사 문제는 한 번에 해결
- **백그라운드 처리**: 무거운 작업은 백그라운드에서 진행

### 제어력 (Control)
- **선택적 동작**: 자동 삭제 없음, 항상 사용자 승인 필요
- **되돌리기**: 가능한 모든 동작에 취소 옵션 제공

## 2. 디자인 시스템

### 2.1 색상 체계

| 역할 | 색상 | 용도 |
|------|------|------|
| Primary | 시스템 블루 | 주요 액션, 선택 상태 |
| Destructive | 시스템 레드 | 삭제, 경고 |
| Success | 시스템 그린 | 완료, 정상 상태 |
| Warning | 시스템 오렌지 | 주의 필요 |
| Neutral | 시스템 그레이 | 비활성, 보조 정보 |

### 2.2 문제 유형별 시각 코드

| 문제 유형 | 아이콘 | 색상 | 심각도 |
|----------|--------|------|--------|
| 다운로드 실패 | `exclamationmark.icloud` | Red | 심각 |
| 손상됨 | `exclamationmark.triangle` | Red | 심각 |
| 다운로드 필요 | `icloud.and.arrow.down` | Orange | 주의 |
| 중복 사진 | `square.on.square` | Blue | 참고 |
| 스크린샷 | `rectangle.on.rectangle` | Gray | 정보 |
| 대용량 | `externaldrive` | Gray | 정보 |

### 2.3 타이포그래피

- **제목**: SF Pro Display (Bold)
- **본문**: SF Pro Text (Regular)
- **숫자**: SF Pro Rounded (Semibold)
- **강조**: SF Pro Text (Semibold)

### 2.4 간격 시스템 (8pt Grid)

enum Spacing {
    static let xs: CGFloat = 4    // 밀접한 요소 간
    static let sm: CGFloat = 8    // 관련 요소 간

## 3. 레이아웃

### 3.1 콘텐츠 영역

```
┌─────────────────────────────────────────┐
│ Safe Area Top                           │
├─────────────────────────────────────────┤
│ ← 16pt →  콘텐츠 영역  ← 16pt →        │
│                                         │
│   화면 가장자리에서 최소 16pt 여백       │
│                                         │
├─────────────────────────────────────────┤
│ Safe Area Bottom                        │
└─────────────────────────────────────────┘
```

### 3.2 정보 위계

```
1순위: 핵심 숫자/상태       → 크기 大, 색상 강조
       예: "127장" 문제 사진 개수

2순위: 카테고리/섹션 제목   → 크기 中, 볼드
       예: "다운로드 필요"

3순위: 설명/상세 정보       → 크기 小, 보조 색상
       예: "iCloud에서 가져와야 하는 사진"

4순위: 메타 정보           → 크기 最小, 회색
       예: "마지막 검사: 2분 전"
```

### 3.3 반응형 그리드

| 환경 | 열 수 | 간격 |
|------|-------|------|
| iPhone (Compact) | 3열 | 2pt |
| iPad (Regular) | 5열 | 4pt |

## 4. 컴포넌트

### 4.1 버튼 스타일

```swift
// Primary: 주요 액션 (파란색 배경, 흰색 텍스트)
// 사용: "스캔 시작", "삭제 확인"

// Secondary: 보조 액션 (테두리만, 파란색 텍스트)
// 사용: "취소", "나중에"

// Destructive: 위험 액션 (빨간색 배경, 흰색 텍스트)
// 사용: "영구 삭제", "모두 삭제"

// Ghost: 최소 강조 (배경 없음, 파란색 텍스트)
// 사용: "더 보기", "건너뛰기"
```

### 4.2 카드 스타일

```
┌─────────────────────────────────────┐
│ 16pt 내부 여백                       │
│  모서리 둥글기: 12pt                 │
│  배경: secondarySystemBackground    │
└─────────────────────────────────────┘
```

### 4.3 리스트 항목

```
┌─────────────────────────────────────┐
│ [아이콘] 제목                    [>] │
│         부제목 (보조 색상)           │
├─────────────────────────────────────┤
│ 최소 높이: 44pt (터치 타겟)          │
└─────────────────────────────────────┘
```

## 5. 인터랙션

### 5.1 제스처

| 제스처 | 동작 | 적용 범위 |
|--------|------|----------|
| 탭 | 선택/실행 | 모든 인터랙티브 요소 |
| 길게 탭 | 선택 모드 진입 | 사진 그리드 |
| 스와이프 좌 | 빠른 삭제 | 리스트 항목 |
| 스와이프 우 | 빠른 복구 | 리스트 항목 |
| 당겨서 새로고침 | 데이터 갱신 | 스크롤 목록 |

### 5.2 피드백

| 동작 | 햅틱 | 시각 |
|------|------|------|
| 선택 | Light | 체크 애니메이션 |
| 삭제 완료 | Success | 토스트 메시지 |
| 오류 | Error | 알림 팝업 |
| 스캔 완료 | Notification | 배지 업데이트 |

### 5.3 삭제 플로우

```
선택 완료 → 확인 대화상자 → 완료 피드백 (햅틱 + 토스트)

┌─────────────────────────────────────┐
│  "3장의 사진을 삭제할까요?"          │
│                                     │
│  이 사진들은 '최근 삭제된 항목'으로   │
│  이동되며, 30일 후 영구 삭제됩니다.  │
│                                     │
│  [취소]              [삭제]         │
└─────────────────────────────────────┘
```

### 5.4 애니메이션

| 유형 | 지속 시간 | 이징 |
|-----|----------|------|
| 마이크로 (체크, 탭) | 0.15초 | easeOut |
| 표준 (화면 전환) | 0.3초 | easeInOut |
| 강조 (주의 환기) | 0.5초 | spring |

## 6. 접근성

### 6.1 필수 요구사항

- **VoiceOver 완벽 지원**: 모든 UI 요소에 의미 있는 레이블
- **Dynamic Type**: 시스템 폰트 크기 설정 존중
- **고대비 모드**: WCAG AA 이상 색상 대비
- **모션 감소**: `reduceMotion` 설정 시 애니메이션 최소화
- **터치 타겟**: 최소 44pt x 44pt

### 6.2 VoiceOver 레이블 예시

```swift
// 삭제 버튼
.accessibilityLabel("선택한 사진 삭제")
.accessibilityHint("15장의 사진을 최근 삭제된 항목으로 이동합니다")

// 상태 배지
.accessibilityLabel("다운로드 필요한 사진 23장")
```

### 6.3 다크모드

```swift
// 절대 금지: 하드코딩 색상
.foregroundColor(.black)  // ❌

// 권장: 시맨틱 색상
.foregroundStyle(.primary)  // ✅
.background(Color(.systemBackground))  // ✅
```

## 7. 성능

### 7.1 반응성 목표

| 동작 | 목표 시간 | 전략 |
|------|----------|------|
| 앱 실행 | < 1초 | 지연 로딩, 캐싱 |
| 사진 목록 스크롤 | 60fps | 이미지 프리페칭, 셀 재사용 |
| 썸네일 로드 | < 100ms | 저해상도 우선 로드 |
| 문제 스캔 | 진행률 표시 | 백그라운드 처리, 증분 업데이트 |

### 7.2 메모리 관리

- 화면에 보이는 이미지만 고해상도 유지
- 스크롤 아웃된 이미지는 자동 해제
- 대용량 작업 시 autorelease pool 활용

## 8. 용어 사전

### 8.1 핵심 용어

| 공식 용어 | 사용 금지 | 설명 |
|----------|----------|------|
| 사진 | 이미지, 에셋, 파일 | 사용자가 저장한 사진 |
| 동영상 | 비디오, 영상 | 사용자가 저장한 동영상 |
| 항목 | 아이템, 에셋 | 사진과 동영상 통칭 |
| 사진 보관함 | 라이브러리, 갤러리 | 전체 사진 저장소 |
| 최근 삭제된 항목 | 휴지통, 쓰레기통 | 삭제 후 복구 가능 영역 |

### 8.2 액션 동사

| 공식 용어 | 사용 금지 | 의미 |
|----------|----------|------|
| 삭제 | 제거, 지우기 | 최근 삭제된 항목으로 이동 |
| 영구 삭제 | 완전 삭제 | 복구 불가능하게 삭제 |
| 검사 | 스캔, 분석 | 사진 보관함 문제 확인 |
| 다운로드 | 가져오기 | iCloud에서 기기로 가져오기 |

### 8.3 상태 표현

| 상태 | UI 표시 |
|-----|---------|
| 대기 중 | "준비 중..." |
| 진행 중 | "검사 중... (35%)" |
| 완료 | "검사 완료" |
| 실패 | "검사 실패" |

### 8.4 수량 표현

| 항목 | 단위 | 예시 |
|-----|------|------|
| 사진 개수 | 장 | "15장" |
| 파일 크기 | KB, MB, GB | "1.2 GB" |
| 시간 | 초, 분, 시간 | "약 2분" |

### 8.5 상대 시간

| 경과 시간 | 표시 |
|----------|------|
| 1분 미만 | "방금" |
| 1-59분 | "{N}분 전" |
| 1-23시간 | "{N}시간 전" |
| 1-6일 | "{N}일 전" |
| 7일 이상 | "YYYY년 M월 D일" |
